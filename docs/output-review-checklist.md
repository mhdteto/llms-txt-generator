# llms.txt Output Review Checklist

Use this checklist before publishing an `llms.txt` file.

## Accuracy

- [ ] The official business name is correct.
- [ ] The location is correct.
- [ ] The business category is accurate.
- [ ] The services listed are actually offered.
- [ ] The target audience is realistic.
- [ ] The website URL is correct.
- [ ] All official links are valid.

## Clarity

- [ ] The description is easy to understand.
- [ ] The file avoids jargon where possible.
- [ ] The structure is readable.
- [ ] The wording is concise.
- [ ] The summary is useful for humans and AI systems.

## Credibility

- [ ] The file does not overclaim.
- [ ] The file does not promise rankings.
- [ ] The file does not claim guaranteed AI citations.
- [ ] The file avoids manipulative instructions.
- [ ] The file does not describe the business as "the best" unless backed by external proof.

## Consistency

- [ ] The information matches the website.
- [ ] The information matches LinkedIn and public profiles.
- [ ] The services match the service pages.
- [ ] The positioning matches the homepage.
- [ ] The contact information is consistent.

## Safety

- [ ] No passwords are included.
- [ ] No API keys are included.
- [ ] No private client information is included.
- [ ] No internal-only documents are linked.
- [ ] No confidential business data is included.

## Publishing

- [ ] The file is saved as `llms.txt`.
- [ ] The file is uploaded to the root of the website.
- [ ] The file is accessible publicly.
- [ ] The file is linked from relevant documentation or resource pages.
- [ ] The file will be reviewed regularly.
